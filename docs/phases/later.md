# Later — Post-MVP Roadmap

This file collects everything intentionally cut from MVP. Items move out of this file into a numbered phase when promoted.

Items are grouped by theme. Order within each group is rough priority.

---

## Canvas extensions

- [ ] **Drill animation / playback** — interpolate between frames; play/pause/scrub
- [ ] **Drill simulation (Stage 2)** — entities with state, timeline, behaviors, events
- [ ] **Free-draw / annotation tool** — Apple-Pencil-style scribble overlay
- [ ] **More primitives** — net, official, bench, goalie crease overlays, defenseman pivot indicator
- [ ] **Path styles** — backwards-skate, screen, deke, body-check
- [ ] **Element groups** — group/ungroup multiple elements
- [ ] **Templates** — start a new drill from a saved template

---

## Practice planner

- [ ] **Per-drill notes** inside a practice
- [ ] **Per-drill duration overrides**
- [ ] **Practice templates** (e.g. pre-game warmup, full-ice systems day)
- [ ] **Block-based practice structure** (warmup / drill / drill / scrimmage)
- [ ] **Print layout** for taped practice sheets

---

## Export and sharing

- [ ] **PDF export** of a practice (multi-page, print-ready)
- [ ] **GIF / MP4 export** once animation lands
- [ ] **Embed code** for a drill in a blog post or coaching forum
- [ ] **OpenGraph drill previews** with rendered thumbnail
- [ ] **Shared drill comments** (coach ↔ player feedback)

---

## Mobile and native

- [ ] **Expo app for iOS and Android** (viewer first, then editor)
- [ ] **Touch-first editing UX** (gesture model, tool palette adapted for thumbs)
- [ ] **Apple Pencil support** for free-draw and precise placement
- [ ] **Offline editing** with sync

---

## Collaboration

- [ ] **Teams, roles, and invites**
- [ ] **Shared drill library per team**
- [ ] **Real-time multiplayer editing**
- [ ] **Coach ↔ assistant coach permissions**

---

## AI and content

> Core AI work (generation, modification, explanation, MCP server, in-app assist) is **Phase 5** — see `docs/phases/5-ai-agents.md`. The items below are AI-adjacent extensions that come **after** Phase 5.

- [ ] **Image-to-drill** — scan a whiteboard photo and convert to a drill (Stage 2)
- [ ] **Voice input** — "add a defender at the right point" via mic
- [ ] **AI behavior generation** — once Stage 2 simulation lands, AI-authored behaviors
- [ ] **AI-suggested drill names and tags** on save
- [ ] **Public drill discovery** — searchable library, ratings, attribution
- [ ] **Drill of the day** content channel
- [ ] **Coach-facing player explanation** generation (post-Phase 5 polish)
- [ ] **Drill variation generator** at scale (curated packs)

## Notes

- Promoting an item into a numbered phase requires:
  1. A user-research signal that this matters to existing users
  2. A scope definition with explicit non-goals
  3. A success metric

- Anything that would compromise the canvas quality bar is rejected, no matter how cool it sounds.
